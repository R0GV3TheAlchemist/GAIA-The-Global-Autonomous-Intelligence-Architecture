## Gaps Identified

### Gap 1 — TaskScheduler Never Started ✅ FIXED
The `TaskScheduler` was instantiated inside every `GAIANRuntime.__init__()` but
its `run_forever()` async loop was never launched at boot or on lazy runtime
creation. Tasks submitted to the queue sat dormant forever. `stats()` would
show `queued: N` permanently.

### Gap 2 — Dual Memory Write Divergence ✅ FIXED
Every conversation turn wrote to two separate, non-communicating memory systems:
- `core/memory/memory_chroma.py` — ChromaDB, legacy, sentence-transformers
- `core/memory/store.py` — SQLite + sqlite-vec, Phase 3, full C17 audit

Memories written by the runtime were never retrieved by the router, and
vice versa. Both systems grew independently forever.

### Gap 3 — ActionGate Not Wired ✅ FIXED
`core/infra/action_gate.py` contains a clean GREEN/YELLOW/RED risk-tier veto
system with a human `confirm_callback` hook. It had never been instantiated,
imported into the request lifecycle, or connected to the Tauri frontend for
RED-tier confirmation dialogs. The `PolicyEngine` in the runtime is a separate
soft gate; the `ActionGate` is the hard infrastructure-level firewall.

### Task 2 — Memory Unification Bridge

**New file:** `core/infra/memory_bridge.py`

Drop-in replacement for `core/memory/memory_chroma.py` convenience helpers.
Exposes identical API:
- `recall_for_prompt(query, gaian_slug, top_k=5) -> list[str]`
- `store_turn(user_message, gaian_response, gaian_slug, session_id, emotion) -> None`

Routes through `GAIANRuntime._memory_store` (SQLite + sqlite-vec, C17-governed)

small signal that synthesised model output has slightly higher recall value.

`kinds` filter on recall: `[MESSAGE, REFLECTION, FACT]` — excludes `GOAL` and
`TASK` items from prompt injection (planner internals, not conversational context).

### Task 3 — ActionGate Full Wiring
**Commit:** [`9df431b`](https://github.com/R0GV3TheAlchemist/GAIA-OS/commit/9df431bed32ec6ce6c12d54f9b670b80bbdd1d83)

**5 files in one atomic commit.**

#### `core/infra/server_state.py` — Singleton
`_action_gate = ActionGate(confirm_callback=None)` instantiated at module load.
`confirm_callback=None` is the safe default: GREEN auto-approves, YELLOW approves
on silence, RED hard-blocks. The real IPC callback is registered in `_startup()`
once the asyncio loop is live (Task 4 completes this).

`get_action_gate() -> ActionGate` accessor exported for all routers.

#### `core/server_state.py` — Stub Updated
Added `_action_gate` and `get_action_gate` to the re-export surface so any
existing caller using the top-level stub continues to work.

#### `core/infra/action_gate_ipc.py` — New
Tauri IPC confirm_callback implementation.

- `make_ipc_confirm_callback()` — factory returning a sync callable for
  `ActionGate(confirm_callback=...)`. Internally drives an `asyncio.Event`
  wait via `ThreadPoolExecutor` — same pattern as `MemoryStore.remember_sync()`.
- Each pending confirmation keyed by UUID4 `request_id` (single-use capability nonce).
- `YELLOW_TIMEOUT = 15s` — proceed on silence. `RED_TIMEOUT = 60s` — block on silence.
- `resolve_pending(request_id, approved)` — called by the resolution router endpoint
  when the frontend POSTs back the human sovereign's decision.
- `_emit_ipc()` — currently emits structured `WARNING` log for Tauri log-listener
  bridge. Single-line swap to real Tauri emit in Task 4 (marked with `# TODO`).

#### `core/routers/chat.py` — Gate Wired in Request Lifecycle
Gate check fires **after** `rt.process()` (engine chain complete, `policy_result`
available) and **before** `_inference_router.stream()` (blocked action never
reaches LLM).

Tier assignment:
- `policy_flagged = True` → `RiskTier.RED`
- Everything else → `RiskTier.GREEN`

This is the safe initial mapping. YELLOW will be assigned to tool-use and
file-write actions when `result.planned_actions` is populated in a future sprint.

If gate blocks: `action_blocked` SSE event fires, stream returns early —
no tokens generated, no memory written.

New `action_gate` field in `done_data`:
```json
"action_gate": {
  "tier":     "green",
  "approved": true,
  "reason":   "Auto-approved: GREEN tier"
}
```

New SSE events emitted:
- `action_gate` — fires when gate approves (tier + approved)
- `action_blocked` — fires when gate denies (reason + tier)

#### `core/routers/action_gate_router.py` — New
Two endpoints:
- `POST /action-gate/respond` — no auth; UUID4 request_id is the capability
  token. Calls `resolve_pending()` to unblock the waiting callback.
- `GET /action-gate/audit` — requires auth; returns full process-lifetime
  audit log from `gate.get_audit_log()`.

## Architecture State — End of Day

```
Request → chat.py
  │
  ├─ _get_runtime(slug)              → GAIANRuntime (cached, scheduler LIVE)
  │     └─ rt.process(message)         → 18 engines + MemoryStore recall
  │                                      PolicyEngine soft-gate fires here
  │
  ├─ ActionGate.evaluate()           → GREEN: pass | YELLOW: surface + wait
  │     └─ if RED or BLOCKED → emit action_blocked SSE, return early
  │     └─ if approved → emit action_gate SSE, continue
  │
  ├─ recall_for_prompt()             → MemoryBridge → MemoryStore (single DB)
  │
  ├─ InferenceRequest(              → GAIAInferenceRouter
  │     gaian_system_prompt=           → T1 Canon
  │     result.system_prompt)          → T2 Criticality
  │                                    → T3 Noosphere
  │                                    → T4 Schumann/BCI
  │                                    → T5 Quintessence
  │                                    → LLM stream → token SSE events
  │
  └─ store_turn()                    → MemoryBridge → MemoryStore (single DB)

ActionGate IPC loop (pending Task 4 — currently log-based):
  gate.evaluate() → _emit_ipc("action-gate-confirm", payload)
                  → asyncio.Event wait (15s YELLOW / 60s RED)
                  ← POST /action-gate/respond {request_id, approved}
                  → resolve_pending() → asyncio.Event.set() → unblock
```
